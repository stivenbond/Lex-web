# Module Specification: AssessmentCreation

## 1. Overview
- **Name**: AssessmentCreation
- **Purpose**: Provide a robust environment for authoring academic assessments, managing question banks, and publishing snapshots for delivery.
- **Schema**: `assessment_creation.*`
- **Primary Stakeholders**: Teachers (Authors), Admins (Templates).

## 2. Domain Boundaries
- **Aggregate Root: Assessment**: Manages a collection of sections and questions organized for a specific evaluation.
- **Aggregate Root: QuestionBank**: A library of reusable questions categorized by subject and difficultly.
- **Entity: Section**: A logical grouping within an assessment (e.g., "Part A: Multiple Choice").
- **Entity: Question**: The fundamental unit of evaluation, supporting multiple types (MCQ, Essay, etc.).

## 3. Module Responsibilities
- Provide a drag-and-drop builder for assessment structure.
- Manage polymorphic question types with type-specific validation.
- Store and categorize questions for future reuse.
- Generate and publish immutable snapshots of assessments for the `AssessmentDelivery` module.

## 4. Integration & Dependencies
- **Inbound Events**: None.
- **Outbound Events**: 
    - `AssessmentPublishedEvent`: Carries the full snapshot payload.
    - `QuestionBankUpdatedEvent`: For reporting/analytics.
- **External Dependencies**: 
    - `ObjectStorage` for image/asset embedding in questions.

## 5. Security & Authorization
- **Permission Constants**: `AssessmentCreationPermissions.cs`
- **Policies**: 
    - `CanManageAssessments`: Authoring rights.
    - `CanManageQuestionBank`: Rights to modify the shared library.

## 6. Cross-Module Interactions
- **AssessmentDelivery**: Consumes snapshots generated by this module.
- **Reporting**: Subscribes to publication events to track curriculum coverage.
- **ObjectStorage**: Used for assets embedded in question text.
